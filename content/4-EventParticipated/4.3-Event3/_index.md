### Overarching Theme

**“DevOps, enterprise operations, cloud architecture, and AWS career development: from real-world work to the AWS Partner ecosystem.”**

### Key Highlights



#### 1. What Does a DevOps Engineer Really Do?

**Speaker:** Trong H. Huong – DevOps Engineer at Bosch Vietnam.

This topic focuses on the real work of a DevOps Engineer, helping attendees understand that DevOps is not simply deploying code or managing servers.

- **Building CI/CD pipelines:** automate build, test, and deployment workflows
- **Infrastructure management and deployment automation:** ensure consistent and efficient system deployment
- **Application containerization:** package and operate applications more flexibly
- **Monitoring and logging:** track system health and detect issues early
- **Ensuring system stability:** maintain reliability and continuous operations

> **Key message:** DevOps acts as a bridge between Development and Operations, making software development, deployment, and operations faster, more automated, and more reliable.



#### 2. Real stories of operations at a multinational corporation

**Speakers:** Mr. Dat Pham – Data Analyst @ Colgate and Mr. Cuong Nguyen – Manufacturing Excellence Engineer.

This topic shares real stories and experiences of operations in a multinational corporate environment.

- **Combining data, technology, and manufacturing processes:** improve business efficiency through operational excellence
- **Data Analyst perspective:** how data is used to support decision-making and optimize processes
- **Manufacturing Excellence Engineer perspective:** solving real-world problems in large enterprises
- **Practical application:** technology and data must address specific operational challenges

> **Key message:** Technology and data only create real value when applied correctly to actual business problems and help improve operational efficiency.



#### 3. A Scalable URL Shortening Service on AWS

**Speakers:** Dinh Trung Kien and Nguyen Minh Tho.

This topic presents how to design a scalable URL shortening system on AWS — a real-world system architecture problem.

- **Generate unique short URLs:** ensure each link is mapped accurately without duplication
- **Fast redirection to original URLs:** optimize response performance for end users
- **Efficient data storage:** design storage mechanisms suited to system scale
- **Scalability:** serve a large number of concurrent users
- **Cloud design thinking:** choose appropriate AWS services, balancing scalability, performance, availability, and cost

> **Key message:** Building a good cloud application is not just about making the system work — it must be designed to scale, operate reliably, and use resources efficiently.



#### 4. From First Cloud AI Journey to AWS Partner

**Speaker:** Dinh Hoang Huu Nghi – AI Engineer at Kyanon Digital and AWS Certified Solutions Architect Associate.

This topic shares the journey from first steps in Cloud and AI to building a career in the AWS ecosystem.

- **Learning AWS:** building foundational cloud knowledge from scratch
- **Real projects:** applying knowledge through hands-on projects
- **Community participation:** learning and exchanging experience within the AWS community
- **Professional certification:** achieving AWS Certified Solutions Architect Associate
- **Toward AWS Partner:** gradually entering the AWS Partner ecosystem

> **Key message:** A career in Cloud and AI requires combining foundational knowledge, hands-on projects, professional certifications, and proactive community participation.
